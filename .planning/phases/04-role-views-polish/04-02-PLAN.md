All admin APIs: verify x-user-role is 'admin' (403 otherwise).

**1. app/api/admin/nodes/route.ts** — POST create node:
- Body: { name, description, branchName?, positionX, positionY, iconKey? }
- Insert into skillNodes, return created node

**2. app/api/admin/nodes/[id]/route.ts** — PUT update + DELETE:
- PUT: update node fields (name, description, branchName, positionX, positionY, iconKey)
- DELETE: delete node (cascade deletes edges and challenges per FK constraints)
- Return updated/deleted confirmation

**3. app/api/admin/edges/route.ts** — POST create edge:
- Body: { sourceNodeId, targetNodeId, requiredMasteryLevel }
- Before inserting: validate no cycle would be created using dag.ts validateDAG
- If cycle detected: return 400 with error message
- Insert into nodeEdges, return created edge

**4. app/api/admin/edges/[id]/route.ts** — DELETE edge:
- Delete edge by id, return confirmation

**5. app/api/admin/challenges/route.ts** — POST create challenge:
- Body: { nodeId, masteryLevel, type, title, description?, content? }
- Insert into challenges, return created challenge

**6. app/api/admin/challenges/[id]/route.ts** — PUT update + DELETE:
- PUT: update challenge fields
- DELETE: delete challenge
- Return confirmation

**7. app/api/admin/analytics/route.ts** — GET analytics:
- totalLearners: count users where role = 'learner'
- activeThisWeek: count users with lastActivityAt in last 7 days (from userNodeMastery)
- avgMasteryPerNode: for each node, average mastery index across learners (use MASTERY_ORDER index)
- popularPaths: nodes ordered by number of learners with mastery records (desc)
- dropOffNodes: nodes where many learners are locked but prereqs should be met (high locked count relative to parent completions)
- Return: { totalLearners, activeThisWeek, nodeStats: [{ nodeId, nodeName, learnerCount, avgMasteryIndex }], topPaths: [...], dropOffs: [...] }
  </action>
