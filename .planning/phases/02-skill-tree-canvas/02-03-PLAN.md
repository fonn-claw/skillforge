**1. lib/tree-utils.ts** — Pure utility functions for unlock computation:
- Export `MASTERY_ORDER` constant: `['locked', 'novice', 'apprentice', 'journeyman', 'expert', 'master'] as const`
- Export `type MasteryLevel = typeof MASTERY_ORDER[number]`
- Export `masteryIndex(level: MasteryLevel): number` — returns index in MASTERY_ORDER
- Export `meetsRequirement(currentLevel: MasteryLevel, requiredLevel: MasteryLevel): boolean` — returns `masteryIndex(currentLevel) >= masteryIndex(requiredLevel)`

- Export `computeNodeUnlockStatus(nodes, edges, masteryMap)`:
  - `nodes`: array of `{ id: string }` (just need ids)
  - `edges`: array of `{ sourceNodeId: string; targetNodeId: string; requiredMasteryLevel: MasteryLevel }`
  - `masteryMap`: `Map<string, MasteryLevel>` — nodeId to current mastery level
  - Returns: `Map<string, { isUnlocked: boolean; shouldBeUnlocked: boolean }>` for each node
  - Logic:
    - For each node, find all incoming edges (where node is the target)
    - If no incoming edges → node is always unlocked (root node)
    - If incoming edges exist → node is unlocked only if ALL source nodes meet their respective requiredMasteryLevel
    - `isUnlocked`: whether the node currently has a mastery record (masteryMap has entry and it's not 'locked')
    - `shouldBeUnlocked`: whether the prerequisite conditions are met (all parents meet required levels)
    - A node where `shouldBeUnlocked=true` but `isUnlocked=false` is a "newly unlockable" node

- Export `computeEdgeStatus(edge, masteryMap)`:
  - Takes a single edge and the mastery map
  - Returns `'inactive' | 'active' | 'completed' | 'unlocking'`
  - `inactive`: source node mastery doesn't meet required level
  - `active`: source meets required level, target is already unlocked (has mastery)
  - `completed`: both source and target have mastery, and source meets requirement
  - `unlocking`: source meets required level but target is still locked (the "just unlocked" state for animation)

**2. components/tree/SkillTreeFlow.tsx** — Integrate unlock logic:

- In the data transformation (toFlowNodes), use computeNodeUnlockStatus to determine if a locked node should actually be unlocked. For nodes where `shouldBeUnlocked=true` and `isUnlocked=false` (the DB says locked but prereqs are met), keep them as 'locked' in the display but add a `canUnlock: true` flag to the node data. This creates the visual distinction for "available to unlock" nodes.
