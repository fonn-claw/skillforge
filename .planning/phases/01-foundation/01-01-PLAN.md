1. Create `db/schema.ts` with the complete schema. ALL of these tables and enums must be present:

   **Enums (using pgEnum):**
   - `roleEnum`: 'learner', 'mentor', 'admin'
   - `masteryLevelEnum`: 'locked', 'novice', 'apprentice', 'journeyman', 'expert', 'master'
   - `challengeTypeEnum`: 'quiz', 'project_submission'
   - `archetypeNameEnum`: 'builder', 'analyst', 'explorer', 'collaborator'

   **Tables:**
   - `archetypes`: id (uuid PK), name (text unique), description (text), color (text for hex), iconKey (text maps to public/assets/icon-archetype-*.svg), createdAt (timestamp)
   - `users`: id (uuid PK), email (text unique), passwordHash (text), displayName (text), role (roleEnum default 'learner'), archetypeId (uuid FK to archetypes nullable), createdAt (timestamp)
   - `skillNodes`: id (uuid PK), name (text), description (text), iconKey (text nullable), positionX (real), positionY (real), branchName (text nullable for 'frontend'/'backend'/'devops'/null), createdAt (timestamp)
   - `nodeEdges`: id (uuid PK), sourceNodeId (uuid FK to skillNodes cascade), targetNodeId (uuid FK to skillNodes cascade), requiredMasteryLevel (masteryLevelEnum default 'novice'), createdAt (timestamp)
   - `challenges`: id (uuid PK), nodeId (uuid FK to skillNodes cascade), masteryLevel (masteryLevelEnum), type (challengeTypeEnum), title (text), description (text nullable), content (jsonb nullable for quiz questions), sortOrder (integer default 0), createdAt (timestamp)
   - `userNodeMastery`: id (uuid PK), userId (uuid FK to users cascade), nodeId (uuid FK to skillNodes cascade), currentLevel (masteryLevelEnum default 'novice'), xpCurrent (integer default 0), xpRequired (integer default 100), unlockedAt (timestamp), lastActivityAt (timestamp), unique index on (userId, nodeId)
   - `challengeSubmissions`: id (uuid PK), userId (uuid FK to users cascade), challengeId (uuid FK to challenges cascade), status (text default 'pending' -- values: pending/passed/failed/in_review), response (jsonb nullable), score (integer nullable), mentorId (uuid FK to users nullable), feedback (text nullable), submittedAt (timestamp), reviewedAt (timestamp nullable)

   Use `drizzle-orm/pg-core` imports: pgTable, pgEnum, uuid, text, integer, real, timestamp, jsonb, uniqueIndex.

2. Create `db/index.ts` with Neon HTTP adapter:
   ```typescript
   import { neon } from '@neondatabase/serverless';
   import { drizzle } from 'drizzle-orm/neon-http';
   import * as schema from './schema';

   const sql = neon(process.env.DATABASE_URL!);
   export const db = drizzle(sql, { schema });
   ```

3. Create `drizzle.config.ts`:
   ```typescript
   import { defineConfig } from 'drizzle-kit';
   export default defineConfig({
     schema: './db/schema.ts',
     out: './db/migrations',
     dialect: 'postgresql',
     dbCredentials: {
       url: process.env.DATABASE_URL!,
     },
   });
   ```

4. Run `npx drizzle-kit push` to push schema to Neon (requires DATABASE_URL in .env.local).
   - If DATABASE_URL is not set yet, note this as a checkpoint -- the user must provide Neon credentials.
   - Verify push succeeds with no errors.
