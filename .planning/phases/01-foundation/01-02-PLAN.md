1. Create `lib/auth.ts` with JWT utilities using jose:
   ```typescript
   import { SignJWT, jwtVerify } from 'jose';

   const secret = new TextEncoder().encode(process.env.SESSION_SECRET!);

   export async function signToken(payload: { userId: string; role: string }) {
     return new SignJWT(payload)
       .setProtectedHeader({ alg: 'HS256' })
       .setExpirationTime('7d')
       .sign(secret);
   }

   export async function verifyToken(token: string) {
     const { payload } = await jwtVerify(token, secret);
     return payload as { userId: string; role: string };
   }
   ```
   Also export cookie helper constants:
   - `SESSION_COOKIE_NAME = 'session'`
   - `COOKIE_OPTIONS = { httpOnly: true, secure: process.env.NODE_ENV === 'production', sameSite: 'lax' as const, path: '/', maxAge: 60 * 60 * 24 * 7 }`

2. Create `lib/validations.ts` with Zod 4 schemas:
   - `loginSchema`: `z.object({ email: z.email(), password: z.string().min(6) })`
   - `registerSchema`: `z.object({ email: z.email(), password: z.string().min(6), displayName: z.string().min(2).max(50) })`
   IMPORTANT: Use Zod 4 syntax -- `z.email()` not `z.string().email()`.

3. Create `middleware.ts` (runs in Edge Runtime -- NO bcryptjs import):
   - Import verifyToken from `./lib/auth`
   - Read `session` cookie from request
   - If no cookie, redirect to `/login`
   - If cookie exists, verify JWT. On failure, redirect to `/login`
   - On success, set `x-user-id` and `x-user-role` headers on the request
   - Matcher config: `['/((?!login|register|api/auth|_next|assets|favicon).*)']`

4. Create `app/api/auth/register/route.ts`:
   - POST handler accepting JSON body `{ email, password, displayName }`
   - Validate with `registerSchema` from lib/validations.ts
   - Check if email already exists in users table (return 409 if so)
   - Hash password with `bcryptjs.hash(password, 10)`
   - Insert new user with role 'learner'
   - Sign JWT with `signToken({ userId: user.id, role: user.role })`
   - Set httpOnly cookie on response
   - Return 201 with `{ id, email, displayName, role }`

5. Create `app/api/auth/login/route.ts`:
   - POST handler accepting JSON body `{ email, password }`
   - Validate with `loginSchema`
   - Look up user by email (return 401 "Invalid credentials" if not found)
   - Compare password with `bcryptjs.compare(password, user.passwordHash)` (return 401 if mismatch)
   - Sign JWT with `signToken({ userId: user.id, role: user.role })`
   - Set httpOnly cookie on response
   - Return 200 with `{ id, email, displayName, role, archetypeId }`

6. Create `app/api/auth/logout/route.ts`:
   - POST handler that clears the session cookie by setting maxAge to 0
   - Return 200 with `{ success: true }`

7. Create `app/api/auth/me/route.ts`:
   - GET handler that reads `x-user-id` header (set by middleware)
   - Queries user from DB by id
   - Returns user object (minus passwordHash) or 401
